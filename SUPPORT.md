# Support

Use [GitHub Discussions](https://github.com/le2ni/zplr/discussions) for integration questions and [GitHub Issues](https://github.com/le2ni/zplr/issues) for reproducible defects. Security reports belong in private advisories as described in `SECURITY.md`.

Include the ZPLr version, runtime/browser, minimal source, options, diagnostics, and expected/actual packed-raster behavior. Physical-printer output is useful context but is not itself proof of a deterministic-renderer defect.

Supported scope is ESM on Node 22/24 and evergreen browsers. CommonJS, Node below 22, physical-printer certification, proprietary font parity, framework wrappers, device/network/RFID side effects, and profiles beyond the October 10, 2025 guide are outside the 0.3/1.0 release sequence.
